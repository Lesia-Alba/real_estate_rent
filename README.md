# Релиз 1.0
## Задача:
Предварительный анализ данных для понимания структуры датасета и качества данных.

## Описание данных:
Датасет содержит объявления об аренде квартир и включает:
- около 24 000 объявлений
- 23 признака
- числовые, категориальные и текстовые поля
- целевую переменную - стоимость аренды
Данные являются сырыми и содержат пропуски, смешанные типы и неструктурированные поля.

# Релиз 2.0
## Задача:
Очистка и подготовка данных.

## Описание данных:
На данном этапе датасет был обработан и приведён к рабочему виду:
- очищены и стандартизированы названия признаков;
- извлечены числовые значения из текстовых полей;
- обработаны пропуски в данных;
- приведены типы данных к корректным форматам.
Итоговый датасет не содержит пропусков и готов для следующего этапа.

# Релиз 2.0
## Задача:
Подготовить финальный датасет для обучения моделей машинного обучения, предсказывающих стоимость аренды квартир в Москве.

##  Что сделано:
- Проведена очистка данных и устранены пропуски.
- Удалены неинформативные и текстовые признаки.
- Все категориальные признаки закодированы в числовой формат.
- Целевая переменная приведена к логарифмическому виду.
- Итоговый датасет не содержит пропусков и дубликатов.

## Основные выводы:
- Стоимость аренды в первую очередь зависит от локации (метро), площади и количества комнат.
- Наличие удобств и инфраструктуры повышает цену аренды.
- Логарифмирование цены снижает влияние выбросов.
